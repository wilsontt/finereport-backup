## 3. 功能需求

### 3.1 憑證輸入

| 功能 | 說明 | 優先級 |
|------|------|--------|
| SSH 連線 | 輸入主機、使用者、密碼，驗證 SSH 連線 | P0 |
| Root 權限 | SSH 成功後輸入 sudo 密碼，驗證 root 權限 | P0 |
| NAS 連線 | 輸入 NAS 主機、ID、PWD、備份路徑，備份時才連線 | P0 |
| 4 碼數字驗證 | SSH、sudo、NAS 驗證完成後，輸入系統顯示的 4 碼數字，防止機器人登入 | P0 |

### 3.2 目錄確認與變更

| 功能 | 說明 | 優先級 |
|------|------|--------|
| 遠端來源目錄 | 顯示探索到的遠端備份目錄，使用者可確認或變更 | P0 |
| NAS 備份目錄 | 顯示 NAS 備份路徑，使用者可確認或變更 | P0 |
| 自訂備份來源 | 可手動新增來源路徑與目的路徑 | P1 |

### 3.3 備份選項

| 功能 | 說明 | 優先級 |
|------|------|--------|
| 刪除舊備份 | 可選，預設不執行；保留期：不執行、3/6/12/24 個月 | P1 |
| 備份月份 | 依執行當月自動產生（如 202503） | P0 |

### 3.4 備份執行

| 功能 | 說明 | 優先級 |
|------|------|--------|
| 開始備份 | 使用者確認後執行備份流程 | P0 |
| 進度顯示 | 畫面上即時顯示備份進度 | P0 |
| 完成報告 | 備份完成後於備份目錄產生 Markdown 報告 | P0 |

## 4. 使用者流程

### 4.1 主流程

```
1. 進入 FineReport 備份工具頁面
2. 輸入 SSH 主機、使用者、密碼 → 點擊「驗證 SSH」
3. 驗證成功後，輸入 sudo 密碼 → 點擊「驗證 Root 權限」
4. 輸入 NAS 主機、ID、PWD、備份路徑 → 點擊「驗證 NAS」
5. 輸入系統顯示的 4 碼數字驗證 → 點擊「驗證」 （防止機器人登入）
6. 點擊「探索遠端目錄」，系統顯示遠端來源與 NAS 路徑
7. 使用者確認或變更目錄（可新增自訂來源）
8. （可選）勾選「刪除舊備份」、選擇保留期
9. 點擊「開始備份」
10. 觀看進度，完成後可下載或檢視報告
```

**技術注意**：若提供 PDF 報告下載，需於 Docker 容器內安裝中文字型（如 Noto Sans CJK TC），否則中文會顯示為方塊或亂碼。

### 4.2 錯誤處理

| 情境 | 預期行為 |
|------|----------|
| SSH 驗證失敗 | 顯示錯誤訊息，不進入下一步 |
| Sudo 驗證失敗 | 顯示錯誤訊息，可重新輸入 |
| NAS 驗證失敗 | 顯示錯誤訊息，可重新輸入 |
| 4 碼數字驗證失敗 | 顯示錯誤訊息，可重新取得驗證碼並輸入 |
| 探索遠端失敗 | 顯示錯誤訊息，提示檢查路徑或手動備份 |
| 備份中斷 | 顯示錯誤訊息，報告已執行步驟 |

## 5. 驗收條件

### 5.1 功能驗收

- [ ] 可輸入 SSH 憑證並驗證連線
- [ ] 可輸入 sudo 密碼並驗證 root 權限
- [ ] 可輸入 NAS 憑證並驗證連線
- [ ] 可完成 4 碼數字驗證（SSH、sudo、NAS 驗證後）
- [x] 可探索遠端備份目錄並顯示
- [x] 可確認或變更遠端來源、NAS 路徑（含編輯、移除、瀏覽 NAS 路徑）
- [x] 可新增自訂備份來源
- [ ] 可選擇刪除舊備份與保留期（預設不執行）
- [ ] 可執行備份並顯示進度
- [ ] 備份完成後產生報告於備份目錄

### 5.2 非功能驗收

- [ ] 部署於企業入口網站，可透過 `/finereport-backup/` 存取
- [ ] 憑證不寫入日誌或持久化儲存
- [ ] 錯誤訊息明確、可理解

### 5.3 技術注意事項

| 項目 | 說明 |
|------|------|
| **PDF 報告中文字型** | 容器內預設無中文字型，若提供 PDF 下載需於 Dockerfile 安裝（如 `fonts-noto-cjk` 或 `ttf-wqy-zenhei`），否則中文顯示為方塊或亂碼 |
